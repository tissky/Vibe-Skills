# VibeSkills v3.0.2 Release Requirement

## Summary

Cut a governed `v3.0.2` release from the current `origin/main` baseline at `c2f98b5`, synchronize the repository-owned release surfaces, and publish a truthful GitHub release that describes the latest merged source rather than the earlier side-release branch used for `v3.0.1`.

## Goal

Publish a new official release for `foryourhealth111-pixel/Vibe-Skills` based on the latest repository source while keeping governed release truth intact.

## Deliverable

- Repository release surfaces updated to `3.0.2` where this repository treats them as governed release markers.
- A detailed release note at `docs/releases/v3.0.2.md`.
- Live release navigation updated to point at `v3.0.2`.
- Runtime/session proof artifacts for this governed run under `outputs/runtime/vibe-sessions/20260413T105521Z-release-v3-0-2-main/`.
- A committed, tagged, pushed, and GitHub-published release if transport and credentials succeed.

## Constraints

- Use the latest remote-backed `main` source at `c2f98b5` as the release base.
- Treat GitHub `v3.0.1` as the latest public release, but remain explicit that it points to side-release commit `d94b772` and is not an ancestor of the current `main` line.
- Prefer the canonical governed release operator for release-owned files instead of hand-editing generated surfaces it owns.
- Do not invent ancestry, migration guarantees, or verification outcomes that the repository cannot prove.
- Preserve existing package metadata behavior unless a release-owned gate or repository convention requires manual changes.
