# 发布 v2.3.54 执行计划

**日期**: 2026-03-30
**需求文档**: [../requirements/2026-03-30-release-v2.3.54.md](../requirements/2026-03-30-release-v2.3.54.md)
**上游规划**: [2026-03-30-non-regression-technical-debt-remediation-plan.md](./2026-03-30-non-regression-technical-debt-remediation-plan.md)
**Internal Grade**: `L`

## Wave Structure

### Wave 1: Freeze Final Release Scope

- 确认 `origin/main` 与当前 worktree 的基线一致。
- 冻结 `v2.3.54` requirement / plan。
- 更新 `config/version-governance.json` 的 release summary，让 release-cut 使用正确摘要。

### Wave 2: Governed Version Cut

- 运行 `scripts/governance/release-cut.ps1 -Version 2.3.54 -Updated 2026-03-30 -Preview`。
- 检查 preview receipt 中的 planned actions 与 sync receipt。
- 运行 apply，更新：
  - version governance
  - maintenance markers
  - changelog / ledger
  - release notes skeleton
  - releases README
  - dist manifests
  - bundled / nested bundled mirrors

### Wave 3: Release Notes Truth Alignment

- 补写 `docs/releases/v2.3.54.md`，使其覆盖本轮已完成且已验证的修复面。
- 只保留有证据的 highlights、validation notes、migration notes。
- 确保无 `TODO`、无重复标题、无缺失质量必需段落。

### Wave 4: Verification

- 运行针对 release / packaging / parity / notes 的 targeted tests 与 gates。
- 如有失败，先修复 truth surface，再重跑。
- 运行 `git diff --check` 作为文本完整性收口。

### Wave 5: Phase Cleanup And Publish Decision

- 运行 node audit / simulated cleanup。
- 清理本轮临时文件与 receipts 以外的无用产物。
- 评估并在权限可用时执行：
  - `git add`
  - `git commit`
  - `git tag`
  - `git push`

## Ownership Boundaries

- Root governed lane:
  - requirement / plan 冻结
  - release summary 与 release notes 真值收口
  - release-cut preview / apply
  - 最终验证、cleanup 与 git 发布
- Mirrors:
  - 仅通过 `sync-bundled-vibe.ps1` / `release-cut.ps1` 派生，不做手工漂移同步

## Delivery Acceptance Plan

- 最小完成标准：
  - `v2.3.54` 的本地 release surface 已完成、已验证、已清理
- 完整发布标准：
  - 在最小完成标准基础上，远端 commit / tag / GitHub release 已成功落地

## Completion Language Rules

- 没有完成 apply + note truth + verification + cleanup 之前，只能说“正在收口 `v2.3.54` 发布面”。
- 没有完成远端对象创建前，只能说“本地发布面已完成”。

## Rollback Rules

- 如果 preview 与 apply 语义不一致，先修 operator，不继续发布。
- 如果 release notes 无法通过质量 gate，不进入 git/tag 步骤。
- 如果远端权限失败，保留本地已验证状态并明确报告阻塞点，不重写版本号。

## Cleanup Expectations

- 阶段结束必须执行 node 审计 / cleanup。
- 清理 `.tmp`、临时 preview receipts 和无用临时目录。
- 不触碰用户未要求回滚的既有脏改动。
