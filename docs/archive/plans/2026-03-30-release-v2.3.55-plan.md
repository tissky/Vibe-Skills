## Wave Structure

### Wave 1: Freeze Release Scope

- 确认当前 worktree 基线是 `origin/main` 的 `f3ab6e5`。
- 冻结 `v2.3.55` requirement / plan。
- 更新 `config/version-governance.json` 的 release summary，让 release-cut 生成正确 README current surface。

### Wave 2: Governed Version Cut

- 运行 `scripts/governance/release-cut.ps1 -Version 2.3.55 -Updated 2026-03-30 -Preview`。
- 检查 preview receipt 和 sync receipt。
- 运行 apply，更新：
  - version governance
  - maintenance markers
  - changelog / ledger
  - release notes skeleton
  - releases README
  - dist manifests
  - bundled / generated compatibility mirrors

### Wave 3: Release Notes Truth Alignment

- 完整补写 `docs/releases/v2.3.55.md`。
- 只写入已经进入 `main` 的修复和已经运行的验证命令。
- 确保没有 TODO、没有重复段落、没有与证据不一致的升级宣称。

### Wave 4: Verification

- 跑本版最小充分验证集：
  - OpenCode / host settings 回归相关测试
  - 卸载 / skill-only host path 相关测试
  - AI config key 分离相关测试
  - macOS shell / bootstrap 兼容相关测试
  - release surface / version gates
- 若有失败，先修 truth surface 或版本面对齐问题，再重跑。

### Wave 5: Publish And Cleanup

- 提交 release commit。
- 推送 `release/v2.3.55`。
- 创建并推送 tag `v2.3.55`。
- 创建 GitHub release。
- 完成 node 审计与临时文件清理。

## Ownership Boundaries

- Root governed lane:
  - requirement / plan 冻结
  - release summary 与 release notes 真值收口
  - release-cut preview / apply
  - 最终验证、git/tag/release 与 cleanup
- Mirrors:

## Delivery Acceptance Plan

- 最小完成标准：
  - `v2.3.55` 的本地 release surface、release notes、版本治理、验证与 cleanup 全部完成
- 完整发布标准：
  - 在最小完成标准基础上，远端 branch、tag 与 GitHub release 已成功创建

## Completion Language Rules

- 没有完成 apply + note truth + verification + cleanup 之前，只能说“正在收口 `v2.3.55` 发布面”。
- 没有完成远端对象创建前，只能说“`v2.3.55` 本地发布面已完成”。

## Rollback Rules

- 如果 preview 与 apply 语义不一致，停止发版，先修 operator。
- 如果 release notes 过不了质量 gate，不进入 tag / GitHub release 步骤。
- 如果远端权限失败，保留本地已验证版本面并明确报告阻塞。

## Cleanup Expectations

- 阶段结束必须执行 node 审计 / cleanup。
- 清理 preview receipt 等临时产物，但保留必要 proof。
- 不触碰与本次发版无关的既有文件。
