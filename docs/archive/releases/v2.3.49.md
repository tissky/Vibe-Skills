# VCO Release v2.3.49

- Date: 2026-03-23
- Commit(base): dc0430d

## Highlights

- Hardened `install.ps1`, `install.sh`, `check.ps1`, and `check.sh` so path derivation no longer breaks when the release worktree lives in shallow locations such as `/tmp/<name>`.
- Added adapter-registry fallback handling to the PowerShell and Python adapter resolvers, allowing installed-runtime health checks to run from `skills/vibe` roots that do not carry repo-level `adapters/index.json`.
- Unified parent-path guarding in shared helpers and install adapters, preventing empty-path failures while keeping canonical-repo gates advisory when only installed artifacts are available.
- Added installed-runtime regression coverage proving the installed `check` surfaces stay operational without repo-level adapter metadata.
