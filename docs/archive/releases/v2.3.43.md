# VCO Release v2.3.43

- Date: 2026-03-15
- Commit(base): c8585d6

## Highlights

- Landed the governed runtime contract as the canonical `vibe` entry surface. The public model is now a single governed path with six fixed stages: `skeleton_check -> deep_interview -> requirement_doc -> xl_plan -> plan_execute -> phase_cleanup`.
- Added the governed runtime configs, templates, and runtime scripts needed to freeze requirement truth before execution while keeping `M/L/XL` as internal execution grades instead of user-facing entry branches.
- Extended the installed-runtime freshness contract to cover the governed runtime surfaces, including runtime configs and runtime scripts, so install-time parity now guards the new entry model instead of treating it as repo-only behavior.
- Added `tests/runtime_neutral/test_governed_runtime_bridge.py` to prove two bridges: the packaging/runtime marker bridge in version governance and the temp artifact-root closure of `invoke-vibe-runtime.ps1`.
- Aligned the outward-facing release surface, install copy, dist manifests, and proof-bundle source-release markers to `v2.3.43`.

## Migration Notes

- `vibe` should now be treated as a governed runtime entry, not as a user-facing grade selector. The public contract is the fixed six-stage state machine documented in `protocols/runtime.md` and `config/runtime-contract.json`.
- `interactive_governed` remains the default assisted mode. `benchmark_autonomous` is the no-follow-up path for benchmark-style or fully delegated execution flows.
- This release does not change Linux public platform truth. Linux governance and promotion truth remain governed by the existing platform-support and proof contracts.
