## Migration Notes

- Operators upgrading from older installs should treat repo pull and installed runtime refresh as separate actions. Pulling `main` does not by itself upgrade `${TARGET_ROOT}/skills/vibe`.
- Proof gates such as `vibe-linux-pwsh-proof-gate.ps1` are canonical repo governance gates. If a locally installed runtime is older than the repo, refresh the installed runtime first and run proof gates from the canonical git checkout.
- This release still does **not** claim Linux `full-authoritative` promotion. The public platform truth remains conservative until the promotion surface, proof bundle, and outward narrative are explicitly recut together.
