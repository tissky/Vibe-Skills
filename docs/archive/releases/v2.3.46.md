# VCO Release v2.3.46

- Date: 2026-03-15
- Commit(base): 63f1973

## Highlights

- Fixed the Linux `benchmark_autonomous` / governed-runtime execution path where benchmark proof units could fail on hosts that expose `python3` but not `python`.
- Introduced a shared Python host resolver in `scripts/common/vibe-governance-helpers.ps1` so governed execution now resolves `${VGO_PYTHON}` through `python`, `python3`, or `py -3` without creating a Linux-only fork.
- Updated `config/benchmark-execution-policy.json`, `scripts/runtime/Invoke-PlanExecute.ps1`, and `scripts/verify/vibe-linux-router-no-pwsh-gate.ps1` to use the host-neutral Python resolver instead of direct `python` calls.
- Extended runtime-neutral bridge tests so host-resolution behavior is now covered at the proof layer, including `python3` fallback and `py -3` launcher prefixing.

## Migration Notes

- Linux operators who previously hit `No such file or directory` from hardcoded `python` inside benchmark/governed runtime should retest on `v2.3.46` before drawing conclusions about Linux lane health.
