- Replaced `USERPROFILE`-only default target-root assumptions across additional official PowerShell entrypoints with the shared platform-aware resolver. - Added execution-context locking to `scripts/verify/vibe-linux-pwsh-proof-gate.ps1`, so running the gate from a stale installed-runtime copy now fails with an explicit governance hint instead of being misread as a live Linux regression in the canonical repo.
- Aligned official-runtime baseline and dist manifests to the governed `2.3.41` release truth, while keeping Linux platform status conservative at `supported-with-constraints` rather than overclaiming promotion.
- Cleaned central operator docs so PowerShell examples now use `pwsh` and `<target-root>` / platform-aware defaults instead of teaching `USERPROFILE`-only invocation patterns.

## Migration Notes

- Linux proof gates are canonical repo governance gates. Run them from the git checkout root, not from `${TARGET_ROOT}/skills/vibe` installed-runtime copies.
- Pulling the repo still does not refresh the installed runtime automatically. Re-run `install.ps1` / `install.sh` or the one-shot bootstrap for the same target root before treating freshness/proof mismatches as runtime regressions.
- Omitting `-TargetRoot` on the updated PowerShell entrypoints now resolves the current-user Codex root through the shared platform-aware resolver. Use `-TargetRoot "<target-root>"` when you need a non-default location.
