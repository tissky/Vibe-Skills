# 发布 v2.3.55 需求

**日期**: 2026-03-30
**目标**: 基于 GitHub 当前最新 `main`（`f3ab6e5`）发布新的稳定版本 `v2.3.55`，让版本面、release notes、tag 与 GitHub release 准确反映 `v2.3.54` 之后已经进入主线的真实修复。

## Intent Contract

- Goal: 切出一个与当前主线真实状态一致的新版本，而不是继续停留在 `v2.3.54` 对后续修复缺失的发布面。
- Deliverable:
  - `v2.3.55` 的仓库版本面更新。
  - `docs/releases/v2.3.55.md` 的正式版本说明。
  - `references/changelog.md`、`references/release-ledger.jsonl`、`docs/releases/README.md`、`dist/*/manifest.json` 的同步更新。
  - 在凭证可用时完成 commit / tag / push / GitHub release。
- Constraints:
  - 必须基于远端最新 `main`，不能从过期分支或未合并分支发版。
  - 发布说明只能陈述已进入 `origin/main` 的改动，不把未合并设计或未来计划写入版本说明。
  - 必须通过现有 release governance 和关键验证 gates，不能用 TODO 占位 release notes。
  - 发版后必须执行阶段清理，包括 node 审计和临时文件清理。
- Acceptance Criteria:
  - `config/version-governance.json`、`SKILL.md`、`bundled/skills/vibe/SKILL.md`、`dist/*/manifest.json` 对齐到 `2.3.55 / 2026-03-30`。
  - `docs/releases/v2.3.55.md` 至少准确覆盖以下已进入主线的修复面：
    - unified owned-only uninstall / skill-only host install-uninstall 对齐
    - OpenCode 启动回归修复与 legacy host settings 兼容
    - intent advice 与 vector diff AI 配置键彻底分离
    - macOS Bash 3.2 / Python 3.10+ bootstrap 先决条件收口
  - release-cut preview 与 apply 一致，README / manifests / ledger / changelog 均由 operator 主路径更新。
  - 针对以上修复面和 release surface 的测试 / gates 通过。
  - 只有在远端 tag 与 GitHub release 真正创建成功后，才允许声称“已发布到 GitHub”。
- Product Acceptance Criteria:
  - 新用户或操作者从 `docs/releases/v2.3.55.md` 能直接理解本版主要解决了什么兼容性 / 配置 / 卸载问题。
  - 当前 release surface、版本治理配置、dist manifests、tag 与 GitHub release 指向同一版本 truth。
- Manual Spot Checks:
  - 检查 `v2.3.54..origin/main` 提交范围，确认 release highlights 没有遗漏实质修复，也没有把纯文档润色夸大成核心功能变更。
  - 检查 OpenCode 与 macOS 相关验证命令覆盖了本版的真实风险点。
  - 检查 `docs/releases/README.md` 已把 `v2.3.55` 设为 current release surface。
  - 检查 git tag 与 GitHub release 的版本号、标题、正文一致。
- Completion Language Policy:
  - 没有完成 release-cut、release note、验证、cleanup 之前，只能说“`v2.3.55` 发布面正在收口”。
  - 没有完成远端 tag / release 创建之前，只能说“`v2.3.55` 本地发布面已完成”。
- Delivery Truth Contract:
  - 本轮完成声明只覆盖 `v2.3.54` 之后已经合入主线并在本次发版中验证过的修复。
  - GitHub 发布状态以实际远端对象创建结果为准，不以本地计划替代事实。
- Non-goals:
  - 不在本轮继续扩展新的 host adapter 或重构安装架构。
  - 不重写历史旧版本 release notes。
  - 不为追求整洁而改动与本次发版无关的既有文件。
- Autonomy Mode: `interactive_governed`
- Inferred Assumptions:
  - 最新已发布 GitHub release 是 `v2.3.54`，因此合理的下一个稳定版本号是 `v2.3.55`。
  - 用户要求的是完整发版，而不是仅本地切版本号，因此在权限允许时应完成 tag 和 GitHub release 创建。
